Adults Data Web API Documentation

Adult

The Adult is sent back and forth as json.
The json format is seen in the example below:
{
    "JobTitle": "Tutor", 
    "ID": 0,
    "FirstName": "Aryan",
    "LastName": "Gifford",
    "HairColor": "Black",
    "EyeColor": "Brown",
    "Age": 56,
    "Weight": 99.2,
    "Height": 172,
    "Sex": "M"
}
API Request Sub-path Description Exceptions
/Adults GET
This request will return an Adult List.

May return 500 Internal Server Error, if something fails.
POST
Use this to create a new Adult. Returns the created Adult. Provide the Adult as json in the request body.
Example:

May return 500 Internal Server Error, if something fails when creating the adult.

DELETE /{id:int}
Use this to delete an Adult. Provide the id for the Adult to delete.
Example:
  • .../Adults/5
  • .../Adults?id=3

May return 204 No content

User

The User is sent back and forth as json.
The json format is seen in the example below:
{
"username":"Alex",
"city":"Horsens",
"age":15,
"password":"123456"
}
API Request Sub-path Description Exceptions
/Users GET /{username:string&password:string}
This request will return an User and validate login.

May return 404 Not Found, if something fails when logging in.
POST
Use this to create a new User. Provide the User as json in the request body.

May return 409 Conflict, if user already exists.
